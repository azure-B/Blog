================================================================================
  STEP 1 — GPU 확인
================================================================================

  딥러닝에서 GPU가 필요한 이유
  ─────────────────────────────
  · 딥러닝은 행렬 연산(Matrix Multiplication)을 수십~수백만 번 반복함
  · CPU : 직렬 처리, 코어 수 적음 (4~16개)   → 학습이 매우 느림
  · GPU : 병렬 처리, 코어 수 매우 많음 (수천 개) → 행렬 연산을 동시에 처리
  · 따라서 GPU가 있으면 학습 속도가 수십~수백 배 빨라짐

  확인 방법
  ─────────
  방법 1) 작업관리자 → 성능 탭 → GPU 항목에서 NVIDIA 제품인지 확인
  방법 2) 제어판 → 시스템 → 시스템 종류에서 "64비트 운영 체제" 확인
  방법 3) 장치관리자 → 디스플레이 어댑터 → NVIDIA GPU 모델 확인

  [중요] CUDA는 NVIDIA GPU에서만 작동함
         AMD GPU는 CUDA 사용 불가 → CPU 버전 TensorFlow만 사용 가능
         64비트 OS가 아니면 설치 불가


================================================================================
  STEP 2 — NVIDIA 그래픽 드라이버 재설치
================================================================================

  · CUDA Toolkit 설치 전에 최신 그래픽 드라이버가 반드시 필요
  · 드라이버 버전과 CUDA 버전의 호환성을 맞춰야 함
  · 드라이버가 오래됐거나 충돌이 있으면 CUDA 설치 후 오류 발생 가능

  설치 방법
  ─────────
  1) https://www.nvidia.com/Download/index.aspx 접속
  2) 자신의 GPU 모델, OS 선택 후 최신 드라이버 다운로드
  3) 기존 드라이버 제거 후 새로 설치 (Clean Installation 권장)


================================================================================
  STEP 3 — CUDA Toolkit 설치
================================================================================

  CUDA란?
  ────────
  · CUDA(Compute Unified Device Architecture)
  · NVIDIA가 만든 GPU 병렬 컴퓨팅 플랫폼 및 API 모델
  · GPU의 가상 명령어셋을 사용할 수 있도록 만들어주는 소프트웨어 계층
  · NVIDIA CUDA 코어가 장착된 GPU에서만 작동
  · 2006년 11월 G80 마이크로아키텍처와 함께 처음 발표
  · 2007년 6월 23일 CUDA SDK 최초 배포

  TensorFlow 1.14와 호환되는 CUDA 버전 : CUDA 10.0
  TensorFlow 2.x와 호환되는 CUDA 버전  : CUDA 11.x

  [핵심] TF 버전 ↔ CUDA 버전 ↔ cuDNN 버전이 모두 맞아야 작동함

  설치 방법
  ─────────
  1) https://developer.nvidia.com/ 접속
  2) CUDA Toolkit 다운로드 페이지 이동
  3) 운영체제: Windows / 아키텍처: x86_64 / 버전: 10 선택
  4) Installer Type: exe(local) 선택 → 다운로드
  5) 설치 파일 실행 → 기본 경로(C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v10.0)에 설치

  설치 확인 방법
  ──────────────
  Anaconda Powershell Prompt(또는 cmd)에서:

    nvcc --version

  결과 예시:
    nvcc: NVIDIA (R) Cuda compiler driver
    Cuda compilation tools, release 10.0, V10.0.130

  ※ nvcc = NVIDIA CUDA Compiler
     이 명령이 작동하면 CUDA가 정상 설치된 것


================================================================================
  STEP 4 — cuDNN 설치
================================================================================

  cuDNN이란?
  ──────────
  · cuDNN(CUDA Deep Neural Network library)
  · NVIDIA가 제공하는 딥러닝 전용 GPU 가속 라이브러리
  · 합성곱(Convolution), 풀링(Pooling), 정규화, 활성화 함수 등
    딥러닝 연산을 GPU에서 최적화하여 처리
  · TensorFlow, PyTorch 등 딥러닝 프레임워크가 내부적으로 사용

  CUDA 10.0에 맞는 cuDNN 버전: cuDNN 7.6.x

  설치 방법
  ─────────
  1) https://developer.nvidia.com/cudnn 접속
  2) NVIDIA Developer 계정 로그인 (무료 가입 필요)
  3) cuDNN 버전 선택 (CUDA 10.0에 맞는 버전)
  4) Windows 10용 zip 파일 다운로드
  5) 압축 해제 후 3개 폴더를 CUDA 설치 폴더에 복사

     복사 경로:
     cuDNN의 bin\        →  C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v10.0\bin\
     cuDNN의 include\    →  C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v10.0\include\
     cuDNN의 lib\x64\    →  C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v10.0\lib\x64\

  [중요] cuDNN은 .exe 설치 파일이 없음 — 파일을 직접 복사해야 함
         파일을 복사하지 않으면 TensorFlow-GPU 실행 시 오류 발생


================================================================================
  STEP 5 — Anaconda 설치
================================================================================

  Anaconda란?
  ───────────
  · Python 과학 계산 특화 배포판 (패키지 관리자 + 환경 관리자)
  · conda 명령으로 패키지 설치 및 가상환경 관리
  · numpy, scipy, matplotlib 등 과학 계산 라이브러리 기본 포함
  · 가상환경(Virtual Environment)으로 프로젝트별 독립 환경 구성 가능

  설치 방법
  ─────────
  1) www.anaconda.com 접속
  2) Windows용 Python 3.x (64-bit) Graphical Installer 다운로드
  3) 설치 실행
     - Install for: Just Me (Recommended) 선택
     - 설치 경로: C:\Users\[사용자명]\anaconda3  (기본값 권장)
     - Advanced Options:
       ☑ Add Anaconda3 to my PATH environment variable  ← 체크 권장
          (PATH에 추가해야 cmd/PowerShell에서 conda 명령 사용 가능)
       ☑ Register Anaconda3 as my default Python

  설치 확인
  ─────────
  Anaconda Powershell Prompt 실행 후:
    conda --version    → conda 버전 출력되면 정상


================================================================================
  STEP 6 — 가상환경 생성 및 TensorFlow-GPU 설치
================================================================================

  가상환경이 필요한 이유
  ──────────────────────
  · 프로젝트마다 필요한 패키지 버전이 다를 수 있음
    (예: TF 1.x 프로젝트와 TF 2.x 프로젝트를 동시에 관리)
  · 가상환경을 분리하면 패키지 버전 충돌 없이 각 프로젝트 독립 실행 가능
  · 환경을 망쳐도 해당 가상환경만 삭제하면 됨 (시스템 영향 없음)

  ┌─────────────────────────────────────────────────────────────────────┐
  │  PPT 기준 3가지 환경 구성 방법                                       │
  ├─────────────────────────────────────────────────────────────────────┤
  │  방법 A) 가상환경: deephive  (TF 1.15, Python 3.6)                  │
  │  방법 B) 가상환경: deephive2 (TF 최신, 한번에 설치)                  │
  │  방법 C) 기본환경: base      (pip으로 직접 설치)                     │
  └─────────────────────────────────────────────────────────────────────┘

  ── 방법 A: deephive 가상환경 (TF 1.15, 시험에서 주로 다루는 환경) ──────

  [1단계] 기본 패키지로 가상환경 생성

    conda create -n deephive python=3.6 numpy scipy matplotlib spyder pandas seaborn scikit-learn pillow h5py

    -n deephive     : 가상환경 이름 = deephive
    python=3.6      : TF 1.x는 Python 3.6과 호환성이 좋음
    numpy scipy     : 수치 계산 라이브러리
    matplotlib      : 그래프 시각화
    spyder          : 과학 계산용 IDE
    pandas          : 데이터 분석
    seaborn         : 통계 시각화
    scikit-learn    : 머신러닝 라이브러리
    pillow          : 이미지 처리
    h5py            : HDF5 파일(모델 저장) 처리

  [2단계] 가상환경 활성화

    conda activate deephive

  [3단계] TensorFlow 설치 — GPU/CPU 버전 선택

    ★ GPU 있을 때 (NVIDIA GPU + CUDA 설치 완료):
      conda install tensorflow-gpu==1.15 keras-gpu

    ★ GPU 없을 때:
      conda install tensorflow==1.15 keras

    [중요] tensorflow-gpu 를 설치하면 CUDA, cuDNN과 연동되어
           GPU를 자동으로 사용함
           cpu 버전은 CUDA 없이도 동작하지만 학습 속도 매우 느림

  [4단계] 추가 패키지 설치

    pip install jupyter notebook opencv-python==4.4.0.46 gym==0.15.7 gym[atari]

    jupyter notebook    : 웹 기반 대화형 개발환경
    opencv-python       : 컴퓨터 비전 라이브러리
    gym                 : OpenAI 강화학습 환경
    버전을 명시하는 이유 : 최신 버전이 TF 1.x와 호환 안 될 수 있음

  ── 방법 B: deephive2 가상환경 (한번에 설치) ───────────────────────────

    conda create -n deephive2 python tensorflow jupyter notebook numpy scipy matplotlib spyder pandas seaborn scikit-learn pillow h5py
    pip install opencv-python gym

    버전을 명시하지 않으면 최신 TF 설치됨 (TF 2.x)

  ── 방법 C: base 환경에 직접 설치 ─────────────────────────────────────

    pip install tensorflow jupyter notebook opencv-python gym

    (가상환경 없이 기본 환경에 설치 — 충돌 위험 있으므로 비권장)

  STEP 7 — TensorFlow 설치 확인

  Anaconda Powershell Prompt에서:

    pip 업그레이드 먼저:

================================================================================
  STEP 8 — Jupyter Notebook 설치 및 실행
================================================================================

  Jupyter Notebook이란?
  ──────────────────────
  · 웹 기반 대화식 개발 환경 (Julia + Python + R 의 합성어 Jupyter)
  · 코드, 수식, 시각화, 텍스트를 한 문서에서 작성·실행 가능
  · .ipynb 파일 형식 사용
  · 딥러닝 실험·교육에 가장 널리 사용

  설치

    pip install jupyter notebook

  실행

    jupyter notebook     ← Anaconda Powershell Prompt에서 실행

    → 브라우저가 자동으로 열리며 http://localhost:8888 접속됨

  사용 방법
  ─────────
  1) 브라우저에서 New → Python 3 클릭 → 새 노트북 생성
  2) 셀에 코드 입력 후 Shift+Enter 로 실행
  3) %load [파일경로]  : 외부 .py 파일을 셀로 불러오기
     예) %load deep_learning_for_everyone[2판]/run_project/py/01_My_First_Deeplearning.py

  종료
  ────
    Ctrl + C  (터미널에서)


================================================================================
  STEP 9 — PyCharm 설치 및 Interpreter 설정
================================================================================

  PyCharm이란?
  ────────────
  · JetBrains사의 Python 전용 통합 개발환경(IDE)
  · 코드 자동완성, 디버깅, Git 연동 등 지원
  · Community Edition(무료)와 Professional Edition(유료)
  · 딥러닝 실습에는 Community Edition으로 충분

  설치 방법
  ─────────
  1) www.jetbrains.com/pycharm 접속
  2) Community 버전 다운로드 (무료)
  3) 설치 실행
     - Create Desktop Shortcut ☑
     - Add "Open Folder as Project" ☑  (탐색기 우클릭 메뉴 추가)
     - .py 파일 연결 ☑ (권장)

  Interpreter 설정 (가장 중요한 부분)
  ─────────────────────────────────────
  · PyCharm이 어떤 Python 환경(conda 가상환경)을 사용할지 지정해야 함
  · 설정하지 않으면 conda 가상환경의 TF, numpy 등을 인식 못 함

  [설정 방법]
  1) PyCharm 실행 → File → Settings (Ctrl+Alt+S)
  2) Project: [프로젝트명] → Python Interpreter 클릭
  3) 우측 상단 톱니바퀴(⚙) → Add...
  4) Conda Environment → Existing environment 선택
  5) Interpreter 경로 지정:
       C:\Users\[사용자명]\anaconda3\envs\deephive\python.exe
       (deephive 가상환경의 python.exe 선택)
  6) OK → Apply

  [확인 방법]
  · Interpreter 목록에 tensorflow, numpy, keras 등 패키지가 보이면 정상
  · 프로젝트 파일(.py)을 열고 Run (Shift+F10) 으로 실행 테스트

================================================================================
  자주 나오는 명령어 정리
================================================================================

  conda 명령어
  ─────────────
    conda create -n [환경명] python=3.6 [패키지...]   : 가상환경 생성
    conda activate [환경명]                           : 가상환경 활성화
    conda deactivate                                 : 가상환경 비활성화
    conda env list                                   : 가상환경 목록 확인
    conda install [패키지명]                          : 패키지 설치
    conda update -n base -c defaults conda           : conda 업데이트

  pip 명령어
  ───────────
    python -m pip install --upgrade pip              : pip 업그레이드
    pip install tensorflow                           : TF 설치 (CPU)
    pip install tensorflow-gpu==1.15                 : TF-GPU 설치
    pip install jupyter notebook                     : Jupyter 설치
    pip install opencv-python==4.4.0.46              : OpenCV 설치
    pip install gym==0.15.7                          : OpenAI Gym 설치

  확인 명령어
  ────────────
