1. 경사하강법과 학습률 (Gradient Descent / Learning Rate)

경사하강법(Gradient Descent)은 손실함수(Loss)를 최소화하기 위해 기울기(Derivative)의 반대 방향으로 가중치를 업데이트하는 최적화 알고리즘이다.
업데이트 공식은 아래와 같다.

W ← W − α · ∇L(W₁, W₂, ...)

여기서 α (alpha)가 바로 학습률(Learning Rate)이다.
α는 한 번의 업데이트에서 가중치를 얼마나 크게 이동시킬지를 결정하는 스텝 크기(Step Size)이다.
α가 크면 크게 이동하고, 작으면 조금 이동한다. 

# TensorFlow 예시 코드 (Lecture 기준)
learning_rate = 0.001
cost = tf.reduce_mean(-tf.reduce_sum(Y * tf.log(hypothesis), reduction_indices=1))
optimizer = tf.train.GradientDescentOptimizer(learning_rate).minimize(cost)

1-1. 학습률이 너무 클 때 — Overshooting / NaN

학습률이 지나치게 크면 한 스텝에 너무 멀리 이동하여 손실이 최솟값을 넘어버린다(Overshooting).
그 결과 손실이 수렴하지 않고 오히려 발산(Diverge)하며, 최종적으로 가중치가 NaN (Not a Number)이 된다.

결과물에서 loss 값이 inf → nan 순서로 출력되면 학습률이 너무 크다는 신호다.

아래는 learning_rate=1.5로 설정했을 때의 실제 출력이다. 

2  cost: 27.2798  W: [[ 0.44... ]]
3  cost: 8.668    W: [[ 0.12... ]]
4  cost: 5.77111  W: [[-0.95... ]]
5  cost: inf      W: [[ nan  nan  nan ]]
6  cost: nan      W: [[ nan  nan  nan ]]
...
Prediction: [0 0 0]
Accuracy:   0.0
왜 NaN이 되는가?
W가 너무 크게 이동 → hypothesis(softmax 값)가 0 또는 1의 극단 → log(0) = -∞ → 손실이 inf → 역전파 기울기도 inf → 가중치가 inf → 컴퓨터는 inf - inf = NaN으로 처리.
즉 수치적 폭발(Numerical Explosion)이 일어나는 것이다.

1-2. 학습률이 너무 작을 때 — Local Minimum / 정체

학습률이 지나치게 작으면 두 가지 문제가 발생한다.
첫째, 수렴하는 데 매우 오랜 시간이 걸린다.
둘째, 기울기가 0에 가까운 구간(지역 최솟값, Local Minimum)에서 완전히 멈춰버린다.

아래는 learning_rate=1e-10으로 설정했을 때의 출력이다. 200 epoch 동안 cost가 전혀 변하지 않는 것을 확인할 수 있다. 

0   cost: 5.73203  W: [[ 0.80... -1.21... ]]
1   cost: 5.73203  W: [[ 0.80... -1.21... ]]
...
200 cost: 5.73203  W: [[ 0.80... -1.21... ]]
Prediction: [0 0 0]
Accuracy:   0.0
왜 업데이트가 안 되는가?
α가 너무 작으면 α · ∇L ≈ 0이 되어 가중치 변화량이 부동소수점(float32) 정밀도 아래로 떨어진다.
컴퓨터 입장에서 W = W - 0.000...001은 그냥 W = W와 동일하게 처리된다.

1-3. 올바른 학습률 선택 전략

정해진 공식은 없으며, 아래 원칙에 따라 실험적으로 찾아야 한다.

전략 내용
여러 값 시도 0.1, 0.01, 0.001 순서로 10배씩 줄이며 시험한다.
Cost 곡선 관찰 에폭마다 cost를 출력·시각화하여 감소 추세를 확인한다.
감소 속도 확인 너무 느리면 올리고, NaN/발산이 보이면 내린다.
통상적인 시작값 0.01 또는 0.001을 출발점으로 삼는 경우가 많다.
Adam, RMSProp 등 적응형 옵티마이저는 학습률을 자동으로 조절해주므로, 실무에서는 이런 방법을 더 많이 쓴다. 하지만 기본 개념은 SGD(GradientDescentOptimizer)의 학습률부터 이해해야 한다.

2. 데이터 전처리 (Data Preprocessing / Normalization)

2-1. 정규화가 필요한 이유 — 왜곡된 등고선

입력 피처(Feature)들의 스케일(scale)이 크게 다를 때 문제가 생긴다.
예를 들어 아래 데이터를 보면 x1은 1~9인 반면 x2는 -5000~9000이다. 

x1   x2      y
1    9000    A
2   -5000    A
4   -2000    B
6    8000    B
9    9000    C

이런 상황에서 손실 함수의 등고선(Contour)이 납작하게 늘어진 타원 형태가 된다.
경사하강법은 기울기 방향으로 이동하는데, 등고선이 찌그러져 있으면 지그재그로 비효율적으로 움직이거나 발산한다.
반면 등고선이 원형(동그란 모양)에 가까울수록 최솟값을 향해 곧장 이동한다.

직관적 비유:
계곡 바닥을 찾아가는 공을 상상하자.
등고선이 납작한 타원 → 공이 좌우로 크게 진동하며 천천히 내려간다.
등고선이 원형 → 공이 가장 가파른 방향 = 바닥 방향으로 직진한다.

2-2. Standardization (표준화) — Zero-centered

표준화는 각 피처를 평균 0, 표준편차 1의 표준 정규분포로 변환하는 방법이다.

x'_j = (x_j − μ_j) / σ_j

x_j : j번째 피처의 원래 값 벡터
μ_j : j번째 피처의 평균 (Mean)
σ_j : j번째 피처의 표준편차 (Standard Deviation)

데이터 분포의 변환 과정은 3단계이다.
Original Data → 평균을 0으로 이동 → Zero-centered Data → 표준편차로 나누어 스케일 통일 → Normalized Data 

# NumPy로 Standardization 구현
import numpy as np

X_std = np.copy(X)
X_std[:, 0] = (X[:, 0] - X[:, 0].mean()) / X[:, 0].std()
X_std[:, 1] = (X[:, 1] - X[:, 1].mean()) / X[:, 1].std()
Standardization 후 효과:
학습률 η = 0.01처럼 큰 값에서도 안정적으로 수렴한다.
(강의 예시: 표준화 후 Adaline이 15 iteration 만에 수렴)

2-3. Min-Max Normalization (최솟값-최댓값 정규화)

각 피처 값을 [0, 1] 범위로 선형 변환하는 방법이다.

x'_j = (x_j − x_min) / (x_max − x_min)

강의 Lab 7-3에서 주가 데이터(x1~x4 범위: 800~1,828,100)를 MinMaxScaler로 변환하였다. 

# MinMaxScaler 직접 구현 (Lab 7-3 참고)
def MinMaxScaler(data):
    numerator   = data - np.min(data, axis=0)
    denominator = np.max(data, axis=0) - np.min(data, axis=0)
    return numerator / (denominator + 1e-7)   # 0으로 나누기 방지

xy = MinMaxScaler(xy)

# 변환 전 (일부)
# [828.66, 833.45, 908100, ...]

# 변환 후 (일부)
# [0.999, 0.999, 0.0,    ...]
# [0.705, 0.704, 1.0,    ...]
# [0.000, 0.077, 0.532,  ...]
방법 수식 결과 범위 특징
Standardization (x − μ) / σ (-∞, +∞), 평균=0 표준편차=1 이상치(Outlier)에 강함
Min-Max (x − min) / (max − min) [0, 1] 이상치에 민감, 범위 보장

3. 과적합 (Overfitting)

3-1. 과적합 정의와 원인

과적합(Overfitting)이란 모델이 훈련 데이터에 지나치게 잘 맞추어져서, 새로운 데이터(테스트 데이터, 실전)에서는 성능이 떨어지는 현상이다.

비유:
시험 문제와 답을 통째로 외웠지만(Memorization), 변형 문제는 전혀 못 푸는 학생과 같다.
훈련 데이터 = 교과서 → 100% 정확도
테스트 데이터 = 실전 시험 → 형편없는 성능

과적합이 발생하는 주요 원인:
훈련 데이터가 너무 적거나, 모델이 너무 복잡하거나(파라미터 수가 많거나), 학습을 너무 오래 진행하거나, 피처(Feature)가 과다할 때 나타난다.

3-2. 과적합 해결책

해결책 설명 장단점
More Training Data
훈련 데이터 추가		 모델이 외울 수 없을 만큼 많은 데이터를 제공한다. 효과 확실. 데이터 수집 비용이 문제.
Reduce Features
피처 수 축소		 불필요하거나 중복되는 입력 변수를 제거한다. 중요한 정보를 버릴 위험이 있다.
Regularization
정규화		 손실 함수에 가중치 크기에 대한 페널티 항을 추가한다. 코드 한 줄로 적용 가능. 실무에서 가장 많이 씀.

3-3. L2 정규화 (Regularization) — 수식 및 코드

정규화의 핵심 아이디어는 "가중치(W)가 너무 커지지 않도록 제한"하는 것이다.
가중치가 크면 모델이 특정 피처에 과도하게 의존하여 훈련 데이터에 과적합된다.

손실 함수에 L2 Regularization 항을 추가한다.

L = (1/N) · Σ D(S(W·xᵢ + b), Lᵢ) + λ · Σ W²

λ (lambda) : 정규화 강도(Regularization Strength)를 조절하는 하이퍼파라미터이다.
λ = 0 이면 정규화 없음, λ = 1 이면 강한 제약, 보통 0.001 정도를 사용한다.
Σ W² : 모든 가중치의 제곱합. 가중치가 클수록 이 항이 커져 손실이 증가하므로 모델이 스스로 W를 작게 유지하려 학습한다. 

# TensorFlow L2 정규화 코드 (Lab 예시)
l2reg = 0.001 * tf.reduce_sum(tf.square(W))
cost  = tf.reduce_mean(-tf.reduce_sum(Y * tf.log(hypothesis), axis=1)) + l2reg
λ 값 가이드:
λ = 0 → 정규화 없음 (과적합 위험)
λ = 0.001 → 약한 제약 (일반적인 시작점)
λ = 1 → 매우 강한 제약 (과소적합 위험)

L1 vs L2:
L1 (Lasso): Σ|W| → 일부 가중치를 완전히 0으로 만들어 피처 선택 효과.
L2 (Ridge): ΣW² → 가중치를 0에 가깝게 줄이지만 완전히 0이 되지는 않음. 일반적으로 더 많이 사용.

4. 학습·검증·테스트 데이터셋 분리

4-1. 왜 Training Set으로 평가하면 안 되는가

훈련 데이터로 학습하고 같은 훈련 데이터로 성능을 측정하면 항상 100% 정확도에 가깝게 나온다.
모델이 답을 외웠기(Memorization) 때문이다. 이는 실제 성능을 전혀 반영하지 못한다.

훈련 데이터 = 교과서(교과서로 공부하고 교과서를 펴놓고 시험을 보는 것과 같다)
테스트 데이터 = 실전 시험지 (처음 보는 문제여야 의미가 있다)

4-2. Training / Validation / Test 3분할

데이터셋은 보통 아래와 같이 3가지로 나눈다.

데이터셋 역할 비유
Training Set
훈련 셋		 모델의 가중치(W, b)를 학습시키는 데 사용한다. 교과서 (공부하는 자료)
Validation Set
검증 셋		 하이퍼파라미터(학습률, 정규화 강도 등)를 튜닝하는 데 사용한다. 학습에는 직접 쓰이지 않는다. 모의고사 (전략 조정용)
Test Set
테스트 셋		 최종 성능 평가에만 사용한다. 학습과 튜닝 모두에 쓰이지 않는다. 실전 시험 (결과만 확인)
Validation Set이 필요한 이유:
Test Set으로 하이퍼파라미터를 튜닝하면 Test Set에도 과적합이 발생한다.
즉 Test Set을 여러 번 보면 그 Test Set에 특화된 모델이 만들어진다.
Validation Set을 따로 두어 튜닝에 쓰고, Test Set은 맨 마지막에 한 번만 사용해야 한다.

4-3. Online Learning

데이터가 너무 많아서 한꺼번에 메모리에 올릴 수 없을 때 사용하는 방법이다.
전체 데이터를 여러 조각(Batch)으로 나누어 순서대로 모델에 먹인다.
강의 예시: 100만 개 데이터를 10만 개씩 나누어 순차 학습.

각 배치로 학습이 끝나면 모델의 가중치가 업데이트된 채로 다음 배치를 받아들인다.
이전 배치에서 학습한 결과가 누적(누적 학습)되므로 MNIST의 next_batch()가 이 방식이다.

5-2. Epoch / Batch Size / Iteration 개념

대용량 데이터를 한 번에 학습시키기 어려우므로 미니배치(Mini-batch) 방식을 사용한다.

용어 정의 예시 (훈련 55,000개, 배치 100개)
Epoch 전체 훈련 데이터를 한 번 모두 통과시키는 것.
1 forward pass + 1 backward pass (전체 데이터 기준).		 15 epoch = 15번 전체 통과
Batch Size 한 번의 forward/backward pass에 사용되는 샘플 수. 100개씩 묶어서 처리
Iteration (= Step) 1 epoch을 완성하는 데 필요한 배치 횟수.
= 전체 데이터 수 ÷ 배치 크기		 55,000 ÷ 100 = 550 iteration
계산 예시:
훈련 데이터 1,000개, 배치 크기 500개
→ 1 epoch = 2 iteration
→ 10 epoch = 20 iteration (총 20번의 가중치 업데이트)

5-3. 전체 TF 코드 + 결과 (Lab 7-4)

from tensorflow.examples.tutorials.mnist import input_data
import tensorflow as tf

mnist    = input_data.read_data_sets("MNIST_data/", one_hot=True)
nb_classes = 10

# 입력 / 출력 플레이스홀더
X = tf.placeholder(tf.float32, [None, 784])    # 28*28 = 784
Y = tf.placeholder(tf.float32, [None, nb_classes])

# 가중치 / 편향
W = tf.Variable(tf.random_normal([784, nb_classes]))
b = tf.Variable(tf.random_normal([nb_classes]))

# 가설 (Softmax)
hypothesis = tf.nn.softmax(tf.matmul(X, W) + b)

# 손실 (Cross Entropy)
cost = tf.reduce_mean(-tf.reduce_sum(Y * tf.log(hypothesis), axis=1))

# 옵티마이저
optimizer = tf.train.GradientDescentOptimizer(learning_rate=0.1).minimize(cost)

# 정확도 계산
is_correct = tf.equal(tf.arg_max(hypothesis, 1), tf.arg_max(Y, 1))
accuracy   = tf.reduce_mean(tf.cast(is_correct, tf.float32))

# 학습 파라미터
training_epochs = 15
batch_size      = 100

with tf.Session() as sess:
    sess.run(tf.global_variables_initializer())

    for epoch in range(training_epochs):
        avg_cost   = 0
        total_batch = int(mnist.train.num_examples / batch_size)  # 550

        for i in range(total_batch):
            batch_xs, batch_ys = mnist.train.next_batch(batch_size)
            c, _ = sess.run([cost, optimizer],
                            feed_dict={X: batch_xs, Y: batch_ys})
            avg_cost += c / total_batch

        print('Epoch:', '%04d' % (epoch + 1),
              'cost =', '{:.9f}'.format(avg_cost))

    print("Learning finished")

    # 테스트셋으로 최종 평가
    print("Accuracy:", accuracy.eval(
        session=sess,
        feed_dict={X: mnist.test.images, Y: mnist.test.labels}
    ))

핵심 키워드 요약 (Ctrl+F 검색용)

Learning Rate / 학습률 / α / alpha — 경사하강법 스텝 크기. 너무 크면 Overshooting/NaN, 너무 작으면 Local Minimum/정체.

Overshooting / 발산 / NaN / inf — 학습률이 너무 클 때 손실이 수렴 못 하고 폭발하는 현상.

Local Minimum / 지역 최솟값 / 정체 — 학습률이 너무 작을 때 기울기가 0에 가까운 구간에서 멈추는 현상.

Preprocessing / 전처리 / Normalization / 정규화 — 피처 스케일 통일을 위한 변환.

Standardization / 표준화 / Zero-centered — (x−μ)/σ, 평균 0 표준편차 1로 변환.

Min-Max / MinMaxScaler — (x−min)/(max−min), [0,1] 범위로 변환.

Overfitting / 과적합 — 훈련 데이터에 과도하게 맞추어 일반화 실패.

Regularization / 정규화(과적합 방지) / L2 / λ / lambda — 손실에 λΣW² 항 추가, 가중치 크기 제한.

l2reg = 0.001 * tf.reduce_sum(tf.square(W)) — TF 구현 코드.

Training Set / Validation Set / Test Set — 3분할 데이터. 훈련/하이퍼파라미터 튜닝/최종 평가 전용.

Epoch / Batch Size / Iteration — Epoch = 전체 1회 통과, Batch = 한 번에 사용하는 샘플 수, Iteration = Epoch 완성에 필요한 배치 수.

Online Learning — 대용량 데이터를 배치로 나누어 순차적으로 학습.

MNIST / 28×28 / 784 / 10 classes — 손글씨 숫자 데이터셋, 입력 784차원, 출력 10클래스.

next_batch(100) — 미니배치 학습에서 100개씩 데이터를 가져오는 메서드.
