LEC 1머신러닝 기본 개념

머신러닝이란? (What is Machine Learning?)

우리가 스팸 필터를 만들어야 한다고 생각해 보자.
수동 프로그래밍으로는 "광고", "무료", "클릭하세요" 같은 단어를 일일이 규칙으로 적어야 한다.
자율 주행도 마찬가지다. 신호등이 빨간색일 때, 사람이 걷고 있을 때, 빗길일 때… 예외가 너무 많아 명시적으로 다 쓸 수 없다.

아서 새뮤얼(Arthur Samuel)은 1959년 이렇게 정의했다.

"Field of study that gives computers the ability to learn without being explicitly programmed"
→ 명시적으로 프로그래밍하지 않아도 스스로 학습하는 능력을 컴퓨터에 부여하는 연구 분야

즉, 규칙을 사람이 하나하나 작성하는 대신 데이터를 보고 컴퓨터가 스스로 규칙을 찾도록 하는 것이 머신러닝의 핵심이다.

지도학습 vs 비지도학습

구분 영문명 정의 예시
지도학습 Supervised Learning 정답 레이블이 있는 데이터로 학습 이미지 분류, 스팸 필터, 시험 점수 예측
비지도학습 Unsupervised Learning 레이블 없는 데이터로 패턴을 스스로 발견 구글 뉴스 그룹화, 단어 클러스터링
지도학습 — Training Set이란? 지도학습에서는 입력(x)과 정답(y)이 쌍으로 묶인 데이터를 학습 데이터(Training Set)라 한다.
예: 이미지 → 고양이 / 강아지 / 머그컵 / 모자로 미리 태그된 사진들이 training set이다.
머신이 이 쌍들을 보고 "이런 특징이면 고양이구나"를 스스로 추출한다.

지도학습의 3가지 문제 유형

지도학습에서 우리가 예측하려는 것이 무엇이냐에 따라 3가지로 나뉜다.

유형 예시 출력 Ctrl+F 키워드
Regression (회귀) 공부 시간으로 시험 점수 예측 연속적인 숫자 (예: 82.5점) regression, 연속값, 점수 예측
Binary Classification (이진 분류) 공부 시간으로 합격/불합격 예측 2개 중 하나 (P/F, 0/1) binary, 이진, 합격 불합격
Multi-label Classification (다중 분류) 공부 시간으로 A/B/C/D/F 학점 예측 여러 클래스 중 하나 multi, 다중, 학점, 분류

Binary: x=10 → P(합격), x=9 → P, x=3 → F, x=2 → F

LAB 1TensorFlow 기초

TensorFlow란?

TensorFlow는 구글이 만든 오픈소스 수치 계산 라이브러리로, 데이터 흐름 그래프(Data Flow Graph)를 사용해 계산을 수행한다.
Python으로 작성하며, 2017년 기준 딥러닝 라이브러리 중 GitHub 지표 1위였다.

Data Flow Graph란? Node(노드): 수학 연산을 나타낸다. (더하기, 곱하기, 활성화 함수 등)
Edge(엣지): 노드 사이를 흐르는 다차원 배열(Tensor)을 나타낸다.
즉, 데이터가 그래프를 따라 흘러가며 변환되는 구조다.

TF Mechanics — TensorFlow의 3단계 동작 원리

TensorFlow는 다음 3단계로 동작한다. 이 순서를 반드시 기억해야 한다.

① Build: TensorFlow 연산(operations)으로 그래프를 먼저 완성한다.
② Feed & Run: sess.run(op, feed_dict={...}) — 데이터를 주입하고 그래프를 실행한다.
③ Update & Return: 그래프 안의 변수들이 업데이트되고 결과값이 반환된다.
비유 — 공장 라인

①은 공장 라인 설계도를 그리는 것이고,
②는 원재료(데이터)를 라인에 올려 기계를 돌리는 것이며,
③은 완성품(결과)이 나오는 것이다.
TensorFlow는 설계도를 먼저 완성하고, 나중에 실제 작업을 수행한다는 것이 핵심이다.

TensorFlow Hello World 코드:

# ① Build — 그래프에 상수 노드 추가
hello = tf.constant("Hello, TensorFlow!")

# ② Feed & Run — 세션을 열고 실행
sess = tf.Session()
print(sess.run(hello))

# 출력: b'Hello, TensorFlow!'
# b'...'에서 b는 Bytes literals를 뜻함

Computational Graph 예시:

# ① Build: 노드 생성
node1 = tf.constant(3.0, tf.float32)
node2 = tf.constant(4.0)          # tf.float32 암묵적 적용
node3 = tf.add(node1, node2)

# ② & ③ Run: 세션 실행
sess = tf.Session()
print(sess.run([node1, node2]))    # [3.0, 4.0]
print(sess.run(node3))             # 7.0
중요 — 왜 sess.run()이 필요한가? TensorFlow에서 node3 = tf.add(node1, node2)를 작성해도 실제 계산은 아직 일어나지 않는다.
이것은 단지 "3.0과 4.0을 더해라"는 설계도를 만든 것이다.
실제 계산은 sess.run(node3)을 호출할 때 비로소 실행된다.
이 지연 실행(Lazy Execution) 방식 덕분에 TF는 최적화, 분산 처리 등을 효율적으로 할 수 있다.

Placeholder — 데이터를 나중에 넣는 슬롯

Variable은 학습 중에 TF가 직접 값을 바꾸는 변수이고,
Placeholder는 그래프 실행 시 외부에서 데이터를 주입받는 빈 슬롯이다. 

a = tf.placeholder(tf.float32)
b = tf.placeholder(tf.float32)
adder_node = a + b    # tf.add(a, b)의 단축 표현

# feed_dict로 실제 값 주입
print(sess.run(adder_node, feed_dict={a: 3, b: 4.5}))
# 출력: 7.5

print(sess.run(adder_node, feed_dict={a: [1, 3], b: [2, 4]}))
# 출력: [3. 7.]
Variable vs Placeholder 비교 Variable: 학습 데이터로 TF가 스스로 조정하는 값 (W, b 같은 가중치)
Placeholder: 우리가 직접 feed_dict로 넣어주는 값 (X, Y 같은 학습 데이터 입력)

쉽게 말하면: Variable은 "기계가 조절하는 손잡이", Placeholder는 "우리가 집어넣는 재료 투입구"다.

LEC 2선형 회귀 (Linear Regression)

가설 H(x) — Hypothesis

데이터를 보면 x(공부 시간)와 y(점수) 사이에 선형적인 패턴이 보인다.
머신러닝은 이 패턴에 가장 잘 맞는 직선을 찾는 것이 목표다.

x (hours)y (score)
1090
980
350
230

선형 회귀에서 우리가 찾으려는 직선의 방정식을 가설(Hypothesis)이라 한다.

$$H(x) = Wx + b$$

$W$ (Weight, 가중치) = 직선의 기울기
$b$ (Bias, 편향) = 직선의 y절편

비유 — 여러 후보 직선

데이터 점들이 찍혀 있을 때, 그 옆을 지나는 직선은 무한히 많다.
파란 선, 노란 선, 빨간 선 중 어떤 것이 데이터를 가장 잘 설명하는가?
이 "가장 잘 맞는 직선"을 찾는 과정이 바로 선형 회귀 학습이다.

비용 함수 cost(W, b) — Cost Function

어떤 직선이 더 나은지 판단하려면 기준이 필요하다.
그 기준이 비용 함수(Cost Function) 또는 손실 함수(Loss Function)다.

직선이 실제 데이터 점과 얼마나 멀리 떨어져 있는지를 측정한다.
예측값 $H(x)$와 실제값 $y$의 차이 $H(x) - y$를 오차(Error)라 한다.

왜 차이를 그냥 더하지 않고 제곱하는가?

제곱을 쓰는 이유 1. 음수 오차와 양수 오차가 서로 상쇄되지 않게 하기 위해
2. 큰 오차에 더 큰 패널티를 주어 극단적인 오류를 더 강하게 줄이기 위해
3. 미분 가능성 — 절댓값과 달리 모든 점에서 미분이 가능해 경사 하강법 적용이 용이함
$$cost(W, b) = \frac{1}{m} \sum_{i=1}^{m} \left(H(x^{(i)}) - y^{(i)}\right)^2$$

$m$ = 학습 데이터의 개수 (샘플 수)
$H(x^{(i)})$ = $i$번째 입력에 대한 우리 모델의 예측값
$y^{(i)}$ = $i$번째 실제 정답값
$\frac{1}{m}\sum$ = 전체 오차의 평균 (Mean Squared Error, MSE)

비유 — 양궁 선수의 점수판

$H(x)$는 내가 쏜 화살의 위치, $y$는 과녁의 중심이다.
cost는 모든 화살이 중심에서 얼마나 벗어났는지의 평균 거리²다.
이 값이 작을수록 내 활(모델)이 좋은 것이다.

목표: Cost를 최소화하라

$$\min_{W, b} \; cost(W, b)$$

결국 머신러닝의 학습 과정은 $W$와 $b$를 조절해 cost를 최소로 만드는 최적의 $W, b$를 찾는 것이다.

LAB 2Linear Regression 코드 구현

TF Mechanics 3단계로 선형 회귀를 구현한다.

① Build — 그래프 구성

import tensorflow as tf

# 학습 데이터 (x: 공부 시간, y: 점수)
x_train = [1, 2, 3]
y_train = [1, 2, 3]

# W, b를 랜덤 정규분포로 초기화 (shape=[1]: 스칼라 1개)
W = tf.Variable(tf.random_normal([1]), name='weight')
b = tf.Variable(tf.random_normal([1]), name='bias')

# 가설: H(x) = Wx + b
hypothesis = x_train * W + b

# 비용 함수: MSE
cost = tf.reduce_mean(tf.square(hypothesis - y_train))

# 최적화: 경사 하강법 (learning_rate=0.01)
optimizer = tf.train.GradientDescentOptimizer(learning_rate=0.01)
train = optimizer.minimize(cost)
tf.reduce_mean 이해하기 t = [1., 2., 3., 4.]일 때 tf.reduce_mean(t)는 2.5를 반환한다.
즉 텐서의 모든 원소 평균을 구하는 함수다. cost 함수의 $\frac{1}{m}\sum$을 이 함수 하나로 처리한다.

② & ③ Run — 세션 실행 및 업데이트

# 세션 시작
sess = tf.Session()

# 전역 변수 초기화 (Variable들을 초기화)
sess.run(tf.global_variables_initializer())

# 2001번 학습
for step in range(2001):
    sess.run(train)
    if step % 20 == 0:
        print(step, sess.run(cost), sess.run(W), sess.run(b))

# 출력 예시:
# 0     2.823   [2.128] [-0.852]
# 20    0.190   [1.533] [-1.050]
# ...
# 2000  1.2e-05 [1.004] [-0.009]  ← W≈1, b≈0 수렴!

Placeholder를 사용한 개선 버전

위 코드는 학습 데이터가 코드에 고정되어 있다.
Placeholder를 쓰면 실행 시점에 다양한 데이터를 유동적으로 넣을 수 있다. 

W = tf.Variable(tf.random_normal([1]), name='weight')
b = tf.Variable(tf.random_normal([1]), name='bias')

# shape=[None]: 데이터 개수에 제한 없음
X = tf.placeholder(tf.float32, shape=[None])
Y = tf.placeholder(tf.float32, shape=[None])

hypothesis = X * W + b
cost = tf.reduce_mean(tf.square(hypothesis - Y))
optimizer = tf.train.GradientDescentOptimizer(learning_rate=0.01)
train = optimizer.minimize(cost)

sess = tf.Session()
sess.run(tf.global_variables_initializer())

for step in range(2001):
    cost_val, W_val, b_val, _ = sess.run(
        [cost, W, b, train],
        feed_dict={X: [1, 2, 3], Y: [1, 2, 3]}   # 이 부분만 바꾸면 다른 데이터 학습 가능
    )
    if step % 20 == 0:
        print(step, cost_val, W_val, b_val)

# 모델 테스트
print(sess.run(hypothesis, feed_dict={X: [5]}))       # [5.011]
print(sess.run(hypothesis, feed_dict={X: [2.5]}))     # [2.500]
print(sess.run(hypothesis, feed_dict={X: [1.5, 3.5]})) # [1.496, 3.504]
shape=[None]의 의미 None은 "아무 숫자나 가능"을 뜻한다.
즉 X에 데이터를 3개 넣어도, 5개 넣어도, 100개 넣어도 동작한다.
배치 크기를 유동적으로 설정할 때 필수적으로 사용된다.

LEC 3Cost 최소화 방법 — Gradient Descent

단순화된 가설과 cost(W) 그래프 모양

b를 제거하고 단순화하면 이해하기 더 쉽다.

$$H(x) = Wx$$
$$cost(W) = \frac{1}{m} \sum_{i=1}^{m} \left(Wx^{(i)} - y^{(i)}\right)^2$$

데이터: x=[1,2,3], y=[1,2,3]일 때 W별 cost를 직접 계산해 보자.

W 값cost(W) 계산결과
W = 1 $\frac{1}{3}((1\cdot1-1)^2 + (1\cdot2-2)^2 + (1\cdot3-3)^2) = \frac{0+0+0}{3}$ 0 (완벽)
W = 0 $\frac{1}{3}((0-1)^2 + (0-2)^2 + (0-3)^2) = \frac{1+4+9}{3}$ 4.67
W = 2 $\frac{1}{3}((2-1)^2 + (4-2)^2 + (6-3)^2) = \frac{1+4+9}{3}$ 4.67

이 값들을 그래프로 그리면 W를 x축, cost(W)를 y축으로 하는 아름다운 U자형(포물선) 그래프가 나온다.
최솟값은 W=1일 때이며, 이것이 우리가 찾는 최적의 W다.

경사 하강법 (Gradient Descent Algorithm)

비유 — 산에서 가장 낮은 곳 찾기

눈을 감고 산에 서 있다고 상상하자.
지금 내 발 아래가 어느 방향으로 기울어져 있는지만 알 수 있다.
그 기울기가 내려가는 방향으로 한 걸음씩 내딛다 보면 결국 가장 낮은 곳(최솟값)에 도달한다.
이것이 경사 하강법의 원리다.

경사 하강법 동작 방식:

1. W를 임의의 값(보통 0 또는 랜덤)으로 시작한다.
2. 현재 위치에서 cost 함수의 기울기(미분값)를 계산한다.
3. 기울기의 반대 방향으로 W를 조금 이동한다.
4. cost가 최솟값에 수렴할 때까지 2~3을 반복한다.

주의: 시작점에 따라 수렴하는 최솟값이 달라질 수 있다.
이를 지역 최솟값(Local Minimum) 문제라 한다.
단, 선형 회귀의 MSE cost 함수는 볼록 함수(Convex Function)이므로 지역 최솟값 = 전역 최솟값이다.
(즉, 선형 회귀에서는 어디서 시작해도 항상 같은 최솟값에 도달한다.)

수학적 공식 유도 — Formal Definition

step 1 — 원래 cost 함수 $$cost(W) = \frac{1}{m} \sum_{i=1}^{m} \left(Wx^{(i)} - y^{(i)}\right)^2$$
↓ 미분 계산을 편하게 하기 위해 분모를 2m으로 변경 (결과는 동일)
step 2 — 분모 2m 버전 $$cost(W) = \frac{1}{2m} \sum_{i=1}^{m} \left(Wx^{(i)} - y^{(i)}\right)^2$$
↓ W에 대해 편미분 (체인룰 적용)
step 3 — 미분 $$\frac{\partial}{\partial W} \frac{1}{2m}\sum_{i=1}^{m}(Wx^{(i)}-y^{(i)})^2 = \frac{1}{m}\sum_{i=1}^{m}(Wx^{(i)}-y^{(i)})x^{(i)}$$
↓ W 업데이트 공식 (알파: 학습률)
최종 — 경사 하강법 업데이트 공식 $$W := W - \alpha \frac{1}{m} \sum_{i=1}^{m} \left(Wx^{(i)} - y^{(i)}\right) x^{(i)}$$
각 기호 해설 $W := W - \alpha \cdot (\text{기울기})$ 형태를 기억하면 된다.
$:=$ : 오른쪽 값을 W에 대입하라 (업데이트하라)
$\alpha$ (알파) : 학습률(Learning Rate) — 한 번에 얼마나 크게 이동할지 결정. 너무 크면 발산, 너무 작으면 느림.
$\frac{1}{m}\sum(Wx^{(i)}-y^{(i)})x^{(i)}$ : cost 함수의 기울기 (미분값). 이 값이 양수면 W를 줄이고, 음수면 W를 늘린다.

왜 분모를 1m에서 2m으로 바꿔도 되나? → 2m은 단지 제곱 미분 시 나오는 계수 2를 약분하기 위한 수학적 편의다. 최솟값 위치는 변하지 않는다.

볼록 함수 (Convex Function) — 선형 회귀가 안전한 이유

cost(W, b)를 3D로 그리면 W와 b가 각각 축이 되고, cost가 높이가 되는 밥그릇 모양(볼록 함수)이 나온다.
볼록 함수는 어느 지점에서 경사 하강법을 시작해도 항상 하나의 전역 최솟값(Global Minimum)으로 수렴한다.
반면 딥러닝에서는 cost 함수가 울퉁불퉁한 고차원 표면이 되어 지역 최솟값 문제가 생긴다.

선형 회귀의 MSE cost 함수는 반드시 볼록 함수(Convex)다.
→ 경사 하강법이 항상 전역 최솟값을 찾아준다는 수학적 보장이 있다.
→ W=5에서 시작해도, W=-3에서 시작해도 모두 같은 W=1로 수렴한다.

LAB 3Gradient Descent 코드 구현

방법 A — 수동으로 기울기 계산

경사 하강법 공식을 그대로 코드로 옮기면 된다.
$W := W - \alpha \cdot \frac{1}{m}\sum(Wx^{(i)}-y^{(i)})x^{(i)}$ 

import tensorflow as tf

x_data = [1, 2, 3]
y_data = [1, 2, 3]

W = tf.Variable(tf.random_normal([1]), name='weight')
X = tf.placeholder(tf.float32)
Y = tf.placeholder(tf.float32)

hypothesis = X * W
cost = tf.reduce_sum(tf.square(hypothesis - Y))

# 기울기 수동 계산: ∂cost/∂W = mean((Wx-y) * x)
learning_rate = 0.1
gradient = tf.reduce_mean((W * X - Y) * X)
descent  = W - learning_rate * gradient
update   = W.assign(descent)   # W에 새 값 대입

sess = tf.Session()
sess.run(tf.global_variables_initializer())

for step in range(21):
    sess.run(update, feed_dict={X: x_data, Y: y_data})
    print(step, sess.run(cost, feed_dict={X: x_data, Y: y_data}), sess.run(W))

# 출력:
# 0   10.0    [?]        ← 랜덤 초기값
# 3   0.133   [1.097]
# 6   0.0003  [1.007]
# 10  2.0e-5  [1.001]
# 20  2.4e-10 [1.0]     ← W = 1.0 에 수렴!

방법 B — TF Optimizer 사용 (추천)

경사 하강법을 직접 구현하는 대신 TF가 제공하는 GradientDescentOptimizer를 쓰면
기울기 계산과 업데이트를 자동으로 처리해 준다. 

# 이 두 줄로 방법 A의 gradient, descent, update 세 줄을 대체!
optimizer = tf.train.GradientDescentOptimizer(learning_rate=0.1)
train     = optimizer.minimize(cost)

# 실행
for step in range(100):
    print(step, sess.run(W))
    sess.run(train)

# W=5로 시작했을 때:
# 0  5.0
# 1  1.266
# 2  1.017
# 3  1.001
# 4  1.000
# 5  1.000   ← 빠르게 수렴

# W=-3으로 시작했을 때도 결국 1.0으로 수렴함 (볼록 함수의 특성)

방법 C — compute_gradients / apply_gradients (심화)

기울기를 한 번 계산하고, 수정한 다음 적용하고 싶다면 분리할 수 있다.
예를 들어 기울기에 클리핑(clipping)을 적용하거나 직접 확인하고 싶을 때 사용한다. 

optimizer = tf.train.GradientDescentOptimizer(learning_rate=0.01)

# 기울기 계산만 (아직 적용 안 함)
gvs = optimizer.compute_gradients(cost, [W])

# 기울기 적용
apply_gradients = optimizer.apply_gradients(gvs)

# 수동 계산 기울기와 TF 계산 기울기가 일치하는지 확인
for step in range(100):
    print(step, sess.run([gradient, W, gvs]))
    sess.run(apply_gradients)

# 출력 예시:
# 0  [37.33, 5.0, [(37.33, 5.0)]]   ← gradient와 gvs가 동일함을 확인
# 1  [33.84, 4.62, ...]
# ...
수동 기울기 vs TF 자동 기울기 — 왜 같은가? 수동으로 계산한 gradient = tf.reduce_mean((W * X - Y) * X)는
$cost = \frac{1}{m}\sum(Wx^{(i)}-y^{(i)})^2$를 W로 직접 미분한 값이다.
TF의 compute_gradients는 자동 미분(Automatic Differentiation)으로 같은 값을 계산한다.
둘이 일치하면 우리 수식이 맞다는 것을 검증한 셈이다.

$$cost(W,b) = \frac{1}{m}\sum_{i=1}^{m}(H(x^{(i)}) - y^{(i)})^2 \quad \text{(MSE 비용 함수)}$$
$$W := W - \alpha \frac{1}{m}\sum_{i=1}^{m}(Wx^{(i)}-y^{(i)})x^{(i)} \quad \text{(경사 하강법 업데이트)}$$

코드 패턴 요약 — 복사 가능

# ━━━ 선형 회귀 전체 패턴 ━━━
import tensorflow as tf

# 1. 데이터
X = tf.placeholder(tf.float32, shape=[None])
Y = tf.placeholder(tf.float32, shape=[None])

# 2. 변수 (학습 대상)
W = tf.Variable(tf.random_normal([1]), name='weight')
b = tf.Variable(tf.random_normal([1]), name='bias')

# 3. 가설
hypothesis = X * W + b

# 4. 비용 함수
cost = tf.reduce_mean(tf.square(hypothesis - Y))

# 5. 옵티마이저
train = tf.train.GradientDescentOptimizer(learning_rate=0.01).minimize(cost)

# 6. 실행
sess = tf.Session()
sess.run(tf.global_variables_initializer())

for step in range(2001):
    _, c, w, b_ = sess.run([train, cost, W, b],
                            feed_dict={X: [1,2,3], Y: [1,2,3]})
    if step % 200 == 0:
        print(f"step={step}, cost={c:.5f}, W={w}, b={b_}")

# 7. 예측
print(sess.run(hypothesis, feed_dict={X: [4, 5]}))
