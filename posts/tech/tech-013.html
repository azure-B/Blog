Algorithm이란?

알고리즘 이란, 문제를 해결하기 위한 일련의 명확하고 유한한 단계이다.
쉽게 말해, 어떤 일을 정확하고 효율적으로 수행하기 위한 방법을 단게별로 정의 한 것이다.

1. Algorithm의 특징

1-1. 명백성

알고리즘의 각 단계는 명확하고 모호하지 않아야 한다.
컴퓨터가 해석 할 수 있는 명확한 명령어로 구성되어야 한다.

예를 들어, 요리레시피를 사람에게는
"소금을 약간 넣고 적당히 끓이세요"
라고 할 수 있지만, 컴퓨터에겐
"소금을 5g 넣고 5분간 끓이세요"
라며 명백하게 말을 해주어야 한다.

왜 명백해야 하냐면,
1. 컴퓨터가 명령이 모호하면 엉뚱한 계산을 하거나, 프로그램이 멈춰버리는 오류가 발생한다.
2. 동일한 알고리즘을 사용하면 A,B 컴퓨터 어디서 실행하든 동일한 결과가 나와야하기 때문이다.

아래는 코드의 예시이다. 


            numbers = [10, 55, 120, 250, 8]

            # '충분히 크다'는 컴퓨터가 이해할 수 없는 모호한 조건입니다.
            for num in numbers:
                if num == "충분히 크면":  # 에러 발생! 기준이 명백하지 않음
                    print("이 숫자는 큽니다:", num)
            
            ---------------------------------------------------------------

            numbers = [10, 55, 120, 250, 8]

            # '100 이상'이라는 명확하고 오해의 소지가 없는 수학적 조건을 제시합니다.
            for num in numbers:
                if num >= 100:  
                    print("이 숫자는 큽니다:", num)

위와 같이 "충분히 크면" 등 모호함을 가지면 안된다.
모든 알고리즘은 명확해야 하며, 컴퓨터가 해석 할 수 있는 명령어야 한다.

1-2. 유한성

알고리즘은 유한한 단계로 구성되어야 한다.
즉, 반복되는 과정이 있더라도 결국은 종료 되어야 한다.
아래는 재귀호출을 이용한 Quick_sort의 예시코드이다. 


            def quick_sort(arr):
                # [유한성을 보장하는 핵심: 종료 조건(Base Case)]
                # 배열의 길이가 1 이하이면, 더 이상 쪼갤 필요 없이 정렬이 끝난 것이므로 그대로 반환
                if len(arr) <= 1:
                    return arr

                # 기준점(pivot) 설정
                pivot = arr[len(arr) // 2]

                # 기준점보다 작은 수, 같은 수, 큰 수로 배열을 분할 (문제의 크기가 점점 작아짐)
                lesser_arr = [x for x in arr if x < pivot]
                equal_arr = [x for x in arr if x == pivot]
                greater_arr = [x for x in arr if x > pivot]

                # 분할된 작은 배열들에 대해 다시 재귀 호출
                return quick_sort(lesser_arr) + equal_arr + quick_sort(greater_arr)


            # 실행 테스트
            numbers = [3, 6, 8, 10, 1, 2, 1]
            print(quick_sort(numbers))
            # 출력: [1, 1, 2, 3, 6, 8, 10]

해당 재귀 호출에서
if len(arr) <= 1를 선언하지 않는다면,
배열이 쪼개질 수 없는 상태가 되어도 끊임없이 재귀호출을 실행하여 메모리의 한계에 부딪혀
RecursionError, Stack Overflow오류가 발생하게 된다.

1-3. 유효성

알고리즘의 각 단계는 컴퓨터에서 수행 할 수 있어야 한다.
연산능력, 메모리 한계 또는 수학적 법칙을 벗어나는 지시를 하면 기계는 수행할 수 없다.
알고리즘을 구성하는 모든 연산은 사람이 계산해 낼 수 있을 만큼 기본적이고 실행 가능해야 한다.

유효성을 위배하는 대표적인 경우

1. 수학적 불가능
0으로 나누기 / 원주율의 마지막 소수점 구하기

2. 무한한 자원 요구
무한대의 크기를 가진 데이터를 메모리에 저장하라는 명령

3. 하드웨어 지원 불가
프린터로 음악을 재생하라

명백성 (Definiteness): "말귀를 오해 없이 알아들을 수 있게 지시했는가?" (표현의 문제)
유효성 (Effectiveness): "그 지시가 실제로 할 수 있는 일인가?" (실행 가능성의 문제)

1-4. 입/출력

알고리즘은 0개 이상의 입력값과 1개 이상의 출력 값을 갖는다.

2. 좋은 Algorithm

지금까지 알고리즘이 반드시 갖춰야 할 조건들에 대해 알아보았다.
결국 알고리즘이란 "입력을 받아, 명백한 지시를 통해, 실행 가능한 연산을 유한한 횟수만큼 반복하여, 원하는 결과를 얻어내는 논리적인 과정"이라고 요약할 수 있다.

이러한 필수 조건들을 완벽하게 지키면서도, 시간과 공간을 가장 적게 사용하는 효율적인 알고리즘을 설계하는 것이 좋은 개발자가 되기 위한 핵심 역량일 것이다.
