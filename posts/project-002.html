프로젝트 개요
기간: 2025.02 — 2025.04 (2개월)
팀: 1인 (개인 프로젝트)
스택: Next.js 14 · Supabase · Tailwind CSS · Vercel

문제 정의

폴라리스 창업동아리를 대표로 맡게 되면서, 멤버 관리·회의록·프로젝트 현황을 한 곳에서 볼 수 있는 내부 툴이 필요했습니다. 기존에는 Notion과 카카오톡을 혼용해 정보가 분산되어 있었습니다.

설계 및 구현

Next.js 14 App Router + Supabase(PostgreSQL)로 빠르게 MVP를 구성했습니다. RLS(Row Level Security)를 적용해 멤버별 권한 분리를 구현했습니다.

-- Supabase RLS 정책 예시
CREATE POLICY "멤버만 읽기 가능"
ON meeting_notes FOR SELECT
USING (
  auth.uid() IN (
    SELECT user_id FROM members
    WHERE club_id = meeting_notes.club_id
  )
);

결과

동아리원 15명이 실제 사용 중이며, 회의록 작성 시간이 평균 30% 단축되었습니다. Vercel 무료 플랜으로 배포해 운영 비용 0원을 달성했습니다.
